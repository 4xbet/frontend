## 2. Технологический стек

- **Vue.js (v3):**
  - Роль: Основной фреймворк для построения пользовательского интерфейса. Обеспечивает реактивность, компонентный подход и управление состоянием.
  - Документация: [https://vuejs.org/](https://vuejs.org/)
  - Рекомендуемые материалы:
    - Официальный туториал: [https://vuejs.org/tutorial/](https://vuejs.org/tutorial/)
    - Vue Mastery (видеокурсы): [https://www.vuemastery.com/](https://www.vuemastery.com/)

- **Vite (v5):**

  - Документация: [https://vitejs.dev/](https://vitejs.dev/)
  - Рекомендуемые материалы:
    - Официальная документация Vite: [https://vitejs.dev/guide/](https://vitejs.dev/guide/)

- **TypeScript (v5):**

  - Документация: [https://www.typescriptlang.org/docs/](https://www.typescriptlang.org/docs/)
  - Рекомендуемые материалы:
    - TypeScript Handbook: [https://www.typescriptlang.org/docs/handbook/intro.html](https://www.typescriptlang.org/docs/handbook/intro.html)

- **Axios:**
  - Роль: Библиотека для выполнения HTTP-запросов к серверу. Используется для взаимодействия с REST API.
  - Документация: [https://axios-http.com/docs/intro](https://axios-http.com/docs/intro)
  - Рекомендуемые материалы:
    - Официальная документация Axios.

- **Pinia (v2):**
  - Роль: Библиотека для управления состоянием приложения. Используется для централизованного хранения и управления данными, доступными по всему приложению.

- **Tailwind CSS:**
  - Роль: Утилитарный CSS-фреймворк для быстрой стилизации компонентов. Позволяет создавать пользовательские интерфейсы, применяя классы непосредственно в HTML.

  - Рекомендуемые материалы:
    - Tailwind CSS: Быстрый старт: [https://tailwindcss.com/docs/installation](https://tailwindcss.com/docs/installation)

- **ESLint & Prettier:**
  - Роль: Инструменты для обеспечения качества кода и единообразного стиля. ESLint помогает находить и исправлять проблемы в коде, а Prettier автоматически форматирует код.

## 3. Структура проекта

- **`/public`**: Содержит статические файлы, которые будут скопированы в корень сборки без изменений (например, `index.html`, `favicon.ico`, изображения).
  - `index.html`: Основной HTML-файл, в который монтируется Vue-приложение.
  - `vite.svg`: Пример статического файла.

- **`/src`**: Основная директория с исходным кодом приложения.
  - `main.ts`: Точка входа в приложение. Здесь инициализируется Vue-приложение, подключаются роутер, Pinia и другие глобальные плагины.
  - `App.vue`: Корневой компонент приложения, обычно содержит `<router-view>` для отображения страниц.
  - `style.css`: Глобальные стили приложения.
  - `assets/`: Директория для статических ресурсов, которые импортируются в код (например, изображения, шрифты). Vite обрабатывает эти файлы и оптимизирует их.
  - `components/`: Содержит переиспользуемые компоненты пользовательского интерфейса.
    - `ui/`: Поддиректория для базовых UI-компонентов (кнопки, инпуты, чекбоксы), которые могут быть стилизованы с помощью Tailwind CSS.
      - `button/`: Компонент кнопки.
      - `input/`: Компонент поля ввода.
      - `checkbox/`: Компонент чекбокса.
    - `TodoForm.vue`: Компонент формы для добавления/редактирования задач.
    - `TodoItem.vue`: Компонент для отображения одной задачи.
  - `lib/`: Вспомогательные утилиты и библиотеки, не связанные напрямую с бизнес-логикой.
    - `utils.ts`: Общие утилиты.
  - `router/`: Конфигурация роутинга приложения.
    - `index.ts`: Определяет маршруты и настройки Vue Router.
  - `stores/`: Содержит хранилища состояния приложения, созданные с помощью Pinia.
    - `todoStore.ts`: Хранилище для управления задачами.
  - `utils/`: Вспомогательные функции и утилиты, специфичные для проекта.
    - `axios.ts`: Конфигурация экземпляра Axios для взаимодействия с API.
  - `views/`: Содержит компоненты, представляющие собой отдельные страницы приложения.
    - `Home.vue`: Главная страница.
    - `About.vue`: Страница "О проекте".
    - `TodoList.vue`: Страница со списком задач.

- **Примеры организации кода:**
  - **Компоненты:** Компоненты должны быть самодостаточными, с четко определенным назначением. Используйте пропсы для передачи данных вниз и события для передачи данных вверх.
  - **Хранилища Pinia:** Разделяйте логику управления состоянием по доменам (например, `todoStore`, `userStore`). Используйте actions для асинхронных операций и mutations (если используются) для синхронных изменений состояния.
  - **Утилиты:** Функции, которые не зависят от состояния приложения и могут быть вызваны из любого места.

## 4. Ключевые концепции и паттерны

- **Управление состоянием (Pinia):**
  - Объяснение: Pinia используется для централизованного управления состоянием приложения. Это позволяет легко обмениваться данными между компонентами и обеспечивает предсказуемость изменений состояния.
  - Основные концепции: Stores, State, Getters, Actions.
  - Документация: [https://pinia.vuejs.org/core-concepts/](https://pinia.vuejs.org/core-concepts/)

- **Роутинг (Vue Router):**
  - Объяснение: Vue Router отвечает за навигацию между различными "страницами" (компонентами-представлениями) в одностраничном приложении.
  - Основные концепции: Маршруты, Динамические сегменты, Навигация.
  - Документация: [https://router.vuejs.org/guide/](https://router.vuejs.org/guide/)

- **Взаимодействие с API (Axios):**
  - Объяснение: Для получения и отправки данных на сервер используется библиотека Axios. Конфигурация клиента Axios находится в `src/utils/axios.ts`.
  - Основные концепции: HTTP-методы (GET, POST, PUT, DELETE), Обработка ответов и ошибок.
  - Документация: [https://axios-http.com/docs/intro](https://axios-http.com/docs/intro)

- **Компонентный подход:**
  - Объяснение: Приложение построено на основе переиспользуемых компонентов Vue. Каждый компонент инкапсулирует свою логику, шаблон и стили.
  - Ключевые моменты: Пропсы (props) для передачи данных вниз, События (events) для передачи данных вверх, Слоты (slots) для композиции.
  - Документация Vue.js: [https://vuejs.org/guide/essentials/component-basics.html](https://vuejs.org/guide/essentials/component-basics.html)

- **Стилизация (Tailwind CSS):**
  - Объяснение: Tailwind CSS используется для быстрой стилизации компонентов с помощью утилитарных классов. Это позволяет избежать написания большого количества CSS-кода и поддерживать единообразный стиль.
  - Ключевые моменты: Утилитарные классы, Конфигурация `tailwind.config.js`.
  - Документация: [https://tailwindcss.com/docs](https://tailwindcss.com/docs)

## 6. Процесс разработки

- **Рабочий процесс:**
  1.  **Создание новой фичи:**
      - Создайте новую ветку в Git от основной ветки разработки (например, `develop` или `main`). Назовите ветку описательно, например, `feature/add-user-profile`.
      - Разработайте новую функциональность, следуя архитектуре проекта и лучшим практикам.
      - Добавьте тесты для новой функциональности.
      - Зафиксируйте изменения (`git commit`).
      - Отправьте ветку на удаленный репозиторий (`git push`).
      - Создайте Pull Request (PR) в основной ветке.
  2.  **Исправление багов:**
      - Если баг обнаружен в продакшене, создайте ветку от основной продакшен-ветки (например, `main` или `master`), например, `hotfix/fix-login-bug`.
      - Если баг обнаружен в процессе разработки, создайте ветку от текущей ветки разработки (`develop`).
      - Исправьте баг, добавьте тесты, зафиксируйте изменения и создайте PR.
  3.  **Ревью кода:**
      - После создания PR, назначьте ревьюеров из команды.
      - Ревьюеры должны внимательно изучить код, оставить комментарии и предложения по улучшению.
      - Автор PR должен внести необходимые правки на основе комментариев.
      - После одобрения PR код будет смержен в основную ветку.

- **Система контроля версий (Git):**
  - **Основные команды:**
    - `git clone <url>`: Клонировать репозиторий.
    - `git checkout -b <branch-name>`: Создать и переключиться на новую ветку.
    - `git add .`: Добавить все измененные файлы в индекс.
    - `git commit -m "Сообщение коммита"`: Зафиксировать изменения.
    - `git push origin <branch-name>`: Отправить ветку на удаленный сервер.
    - `git pull origin <branch-name>`: Получить изменения с удаленного сервера.
    - `git merge <branch-name>`: Слить другую ветку в текущую.
  - **Ветвление:** Используется стратегия Gitflow или упрощенная версия с ветками `main` (продакшен) и `develop` (разработка). Для новых фич и исправлений создаются отдельные тематические ветки.
  - **Pull Requests (PR):** Используются для ревью кода перед слиянием в основную ветку.

- **CI/CD (Непрерывная интеграция / Непрерывное развертывание):**
  - (Если применимо) Описание пайплайна: Например, "При создании PR запускаются автоматические тесты и линтеры. После слияния в `develop` происходит деплой на staging-сервер. После слияния в `main` происходит деплой на продакшен."
  - Инструменты: GitHub Actions, GitLab CI, Jenkins и т.д.

## 7. Часто задаваемые вопросы (FAQ)

- **Q: Как мне добавить новый компонент?**
  - A: Создайте новый `.vue` файл в директории `src/components/` (или в поддиректории `ui/`, если это базовый UI-элемент). Определите его шаблон, скрипт и стили. Если компонент будет использоваться на нескольких страницах, зарегистрируйте его глобально в `main.ts` или импортируйте локально в нужных компонентах/страницах.

- **Q: Где хранятся данные приложения?**
  - A: В текущей конфигурации данные задач хранятся в `todoStore.ts` с использованием Pinia. Для персистентности данных может использоваться локальное хранилище браузера (localStorage) или синхронизация с бэкендом через API.

- **Q: Как обратиться к API?**
  - A: Используйте экземпляр `axios` из `src/utils/axios.ts`. Например:

    ```typescript
    import axios from '@/utils/axios';

    async function fetchTasks() {
      try {
        const response = await axios.get('/tasks');
        return response.data;
      } catch (error) {
        console.error('Ошибка при получении задач:', error);
        throw error;
      }
    }
    ```

- **Q: Как добавить новую страницу (маршрут)?**
  - A: Создайте новый Vue-компонент для вашей страницы в директории `src/views/`. Затем добавьте новый маршрут в `src/router/index.ts`, указав путь и компонент для этого маршрута.

- **Q: Где найти документацию по Vue.js / Pinia / Tailwind CSS?**
  - A: Ссылки на официальную документацию и рекомендуемые материалы приведены в разделе "Технологический стек".
